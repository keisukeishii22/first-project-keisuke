# 自動運用ルーティーン

このプロジェクトは「指示がなくても週次でコンテンツを生成する」ルーティーン運用を行う。

## ルーティーンの中身(毎週日曜に実行)

1. `進捗メモ.md` の「生成ログ」を読み、**まだ生成していない最若番の週**を特定する。
2. 「12週テーママップ.md」の該当週の指定と「共通ルール」を読み込む。
3. その週のインタビュー回答(圭佑からの3問回答)があれば反映する。なければ **素材待ちドラフト**(テーママップ指定のみで構成・本文骨子まで作成し、実体験の挿入箇所を `【ここに実体験:〇〇】` で明示)として生成する。
4. 次の4点を一括生成し、`生成物/第N週/` 配下に保存する:
   - `note記事.md`(タイトル案3本付き)
   - `リール台本.md`(2本、各30〜45秒。フック→本編→CTA)
   - `カルーセル構成.md`(表紙+5枚+CTA)
   - `ストーリーズ告知.md`
5. `進捗メモ.md` の生成ログに「第N週パック生成済み(日付)」を追記する。
6. ブランチ `claude/jolly-faraday-ay5xjo` にコミット&プッシュする。
7. 圭佑に「第N週パックを生成・保存しました。投稿前にファクトチェックと実体験の追記をお願いします」と報告する。

## KPI ルーティーン

KPI(フォロワー数・保存数・プロフィールアクセス・note ビュー・LINE 登録数)を報告されたら:

1. `進捗メモ.md` の「KPI記録欄」に1行追記する。
2. 保存数上位テーマを「振り返りメモ」に記録し、翌週以降の生成方針(フック・切り口)に反映する。

## 「指示がなくても動く」ための起動方法(2通り)

このルーティーンを **圭佑からの指示なしで** 毎週走らせるには、次のいずれかを設定する:

### 方法A:Claude Code on the web の「スケジュール実行(Scheduled sessions / triggers)」(推奨)

Claude Code on the web の管理画面で、このリポジトリに対して週次トリガーを作成し、プロンプトに次を設定する:

> 「自動運用ルーティーン.md に従って今週分のテーマパックを生成し、コミット&プッシュして報告して」

- 推奨スケジュール:**毎週日曜 朝**(JST)。
- 設定手順の詳細:https://code.claude.com/docs/en/claude-code-on-the-web

### 方法B:現セッション内での定期実行

このセッションを開いたまま運用する場合は、`/loop` で定期実行できる(例:`/loop 7d 自動運用ルーティーン.md に従って次週分を生成`)。ただしコンテナは無操作が続くと回収されるため、継続運用には方法Aを推奨する。

## ガードレール(必ず守る)

- 生成物は **下書き**。最終投稿は必ず圭佑の確認(ファクトチェック・実体験の追記)を経る。
- 補助金・法令・税制・許可制度に触れる週(第4・5・7・10週)は、数値や要件を断定せず「公開前に一次情報で確認」と注記する。
- 取引先・契約内容の固有名詞は出さない。現場が特定できる写真指定はしない。
