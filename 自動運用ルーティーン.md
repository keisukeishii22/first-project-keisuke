# 自動運用ルーティーン

このプロジェクトは「指示がなくても週次でコンテンツを生成する」ルーティーン運用を行う。

## ルーティーンの中身(毎週日曜に実行)

1. `進捗メモ.md` の「生成ログ」を読み、**まだ生成していない最若番の週**を特定する。
2. 「12週テーママップ.md」の該当週の指定と「共通ルール」、および「情報源ガイドライン.md」を読み込む。
3. その週のインタビュー回答(圭佑からの3問回答)があれば反映する。なければ **素材待ちドラフト**(テーママップ指定のみで構成・本文骨子まで作成し、実体験の挿入箇所を `【ここに実体験:〇〇】` で明示)として生成する。
   - 制度・法令・数字を扱う週(第4・5・7・10週など)は、Web 検索が使えるセッションなら生成前に一次情報(国交省・e-Gov 等)を検索・参照し、note 記事の「参考情報源」欄に実 URL を記載する。検索できない経路では発行元と確認ポイントの列挙にとどめ、数値を断定しない。
4. 次の4点を一括生成し、`生成物/第N週/` 配下に保存する:
   - `note記事.md`(タイトル案3本+参考情報源欄付き。[検索型]の週はテーママップ共通ルールの「SEO ブロック」も含める)
   - `リール台本.md`(2本、各30〜45秒。フック→本編→CTA)
   - `カルーセル構成.md`(表紙+5枚+CTA)
   - `ストーリーズ告知.md`
5. `進捗メモ.md` の生成ログに「第N週パック生成済み(日付)」を追記する。
6. そのセッションの作業ブランチ(セッション開始時に指定される `claude/...` ブランチ)にコミット&プッシュする。
7. 圭佑に「第N週パックを生成・保存しました。投稿前にファクトチェックと実体験の追記をお願いします」と報告する。

## KPI ルーティーン

KPI(フォロワー数・保存数・プロフィールアクセス・note ビュー・LINE 登録数)を報告されたら:

1. `進捗メモ.md` の「KPI記録欄」に1行追記する。
2. 保存数上位テーマを「振り返りメモ」に記録し、翌週以降の生成方針(フック・切り口)に反映する。

## 「指示がなくても動く」ための起動方法

### 正式経路:Claude Code on the web の「スケジュール実行(Scheduled sessions / triggers)」

**2026-07-02 に GitHub Actions の定期実行から移行**。理由:Claude セッションは Web 検索が使えるため、制度・法令・数字を生成前に一次情報で裏取りし、実 URL 付きの「参考情報源」欄を付けられる(API スクリプト経路では不可能)。

設定手順(圭佑が1回だけ実施):

1. Claude Code on the web(claude.ai/code)でこのリポジトリ(keisukeishii22/first-project-keisuke)を開く
2. スケジュール実行(Scheduled sessions / triggers)で週次トリガーを作成
3. スケジュール:**毎週日曜 朝(JST)**
4. プロンプトに次を設定:

> 「自動運用ルーティーン.md に従って今週分のテーマパックを生成し、コミット&プッシュして報告して。制度・法令・数字を扱う週は生成前に Web 検索で一次情報を裏取りし、参考情報源欄に実 URL を記載すること」

- 設定手順の詳細:https://code.claude.com/docs/en/claude-code-on-the-web

### フォールバック:GitHub Actions の手動実行

`.github/workflows/weekly-content.yml` は手動実行(workflow_dispatch)専用として残してある。スケジュールセッションが使えない緊急時のみ、GitHub の Actions タブから手動実行する。この経路は Web 検索ができないため、参考情報源は「確認先の列挙」にとどまる点に注意。

## ガードレール(必ず守る)

- 生成物は **下書き**。最終投稿は必ず圭佑の確認(ファクトチェック・実体験の追記)を経る。
- 補助金・法令・税制・許可制度に触れる週(第4・5・7・10週)は、数値や要件を断定せず「公開前に一次情報で確認」と注記する。
- 情報源の選定・引用形式・ファクトチェック手順は「情報源ガイドライン.md」に従う。出典を示せない数字は書かない。
- 取引先・契約内容の固有名詞は出さない。現場が特定できる写真指定はしない。
- **自動起動は1経路に絞る**:GitHub Actions(weekly-content.yml)と Claude のスケジュール実行を併用すると同じ週を二重生成しかねない。定期実行は Claude スケジュールセッションのみとし、GitHub Actions は手動フォールバック専用(cron は 2026-07-02 停止済み)。
- **未公開ドラフトが3週分を超えている間は、新規生成よりインタビュー回答の取得と公開を優先する**(進捗メモ.md の公開状況を参照)。
