# 情報源・引用ガイドライン

全生成物(note・リール・カルーセル・ストーリーズ、および今後の HP ブログ・石善建設 / 圭佑不動産の各アカウント)に適用する。目的は2つ:**事実の齟齬をなくすこと**と、**元ネタの幅を広げてコンテンツの深みを増すこと**。

## 情報源の3層ルール

### 層1:一次情報(事実・数字・制度の根拠には必須)

法令・制度・税制・補助金・統計の数値は、必ずこの層で裏取りしてから公開する。

| 分野 | 確認先 |
|------|--------|
| 法令の条文(建設業法・施行令など) | e-Gov 法令検索 https://laws.e-gov.go.jp/ |
| 建設業許可・技術者制度・事業承継 | 国土交通省 https://www.mlit.go.jp/(不動産・建設経済局) |
| 千葉県知事許可の運用・手引き | 千葉県 建設業許可担当ページ |
| 税制 | 国税庁 https://www.nta.go.jp/ |
| 補助金・支援制度 | 中小企業庁・ミラサポplus・こどもエコすまい等の各公式サイト |
| 住宅ローン・金利 | 住宅金融支援機構 https://www.jhf.go.jp/ |
| 資格試験の制度・日程 | 各試験実施機関(建設業振興基金・建設技術教育センター等)の公式サイト |
| 統計(着工数・業界規模など) | e-Stat https://www.e-stat.go.jp/ 、国交省「住宅着工統計」「建設業構造実態調査」 |

### 層2:信頼できる二次情報(補強・データ・研究)

記事に厚みを出すために積極的に使ってよい。引用時は出典名を明記する。

- **学術論文**:J-STAGE https://www.jstage.jst.go.jp/ 、CiNii、Google Scholar(日本建築学会・土木学会の論文など)
- **業界団体**:全国建設業協会、日本建設業連合会、全国工務店協会(JBN)など
- **業界メディア**:新建ハウジング、日経クロステック(住宅・建設)など
- **公的白書**:国土交通白書、中小企業白書(事業承継まわりのデータが豊富)

### 層3:着想・切り口用(事実の根拠には使わない)

- YouTube(経営系・建設系チャンネル。例:中野優作氏など)
- 他社のブログ・note・SNS 投稿

この層は**フック・構成・切り口の参考**として広く見る。ただし、ここで得た数字や制度の説明をそのまま書かない。使う場合は層1で照合し、考え方・言葉を引用するときは「〜氏が YouTube で語っていた」等と出典を明記する。

## 引用・記載ルール

1. **note 記事の末尾に「参考情報源」欄を必ず設ける。** 記載形式:`発行元「ページ/資料名」URL(閲覧日)`。
2. 法令に触れるときは条文番号まで書く(例:建設業法第3条)。条文番号が不明なら断定せず「一次情報で確認」の注記を残す。
3. **出典を示せない数字は書かない。** AI が記憶ベースで出した数値・金額基準・年数は、必ず層1で照合してから確定する(制度改正で古くなっている前提で扱う)。
4. 他者(人物・企業・チャンネル)への言及は、批判にならない形+出典明記。契約・取引に関わる相手は従来どおり匿名化。
5. リール・カルーセルは尺の都合で出典を載せられないため、**根拠は連動する note 記事側に集約**し、動画・画像側は「詳しくは note で」で誘導する。

## 公開前ファクトチェック手順(圭佑のチェックリスト)

1. 本文中の数字・金額・年数・条文をマーカーで拾い出す
2. それぞれ「参考情報源」欄の URL で現行の内容と一致するか確認する(改正日にも注意)
3. 体験談部分が事実と相違ないか本人確認、守秘条項・関係者の掲載許可を確認
4. 確認が取れない記述は削除するか「〜と言われている」に弱める

## 週別の重点確認先

- **第4週(資格)**:試験実施機関の公式サイト(受験資格・日程・制度改正)
- **第5週(建設業許可)**:e-Gov(建設業法・施行令)+千葉県の手引き ※軽微な工事の金額基準・特定建設業の下請代金基準は近年改正があるため必ず現行条文で確認
- **第7週(下請け契約)**:建設業法(19条ほか)・国交省「建設工事標準下請契約約款」
- **第10週(事業承継)**:国交省の事業承継関連通知+中小企業庁「事業承継ガイドライン」

## Claude(生成側)の動き方

- **Web 検索が使えるセッション**(Claude Code のスケジュール実行・対話セッション)では、制度・数字に触れる週は生成前に層1の該当ページを検索・参照し、実 URL を「参考情報源」欄に記載する。
- **Web 検索が使えない経路**(GitHub Actions の API スクリプト)では、数値を断定せず「公開前に一次情報で確認」注記+「参考情報源(確認先)」として発行元と確認ポイントを列挙する。
- 層3(YouTube・ブログ)からフックや構成の型を借りたときは、制作メモに参考元を記録しておく(パクリ防止と、後で出典を明示できるようにするため)。
