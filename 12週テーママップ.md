# 12週テーマ×4展開マップ

## 使い方

毎週日曜、Claude Code に「第N週のテーマパックを生成」と指示するだけで、note 記事下書き・リール台本2本・カルーセル構成案・ストーリーズ告知文を一括生成する。生成物は必ず圭佑の確認(ファクトチェック・実体験の追記)を経てから投稿する。

## 記事の役割タグ

- **[送客型]**:物語系。Instagram 送客＋note 編集部の注目記事狙い。タイトルは感情と数字で引く。
- **[検索型]**:実務系。Google 検索流入狙い。タイトルに検索キーワードを必ず含める。

## 共通ルール

- リールの締めは毎回「続きは note で」または「詳しくはプロフィールのリンクから」。
- カルーセルは個人アカウント用(石善建設アカウントの自動カルーセルとは別系統)。
- アフィリリンクを含む場合は冒頭に PR 表記(フェーズ1は物販のみ)。
- 補助金・法令・税制は公開前に一次情報で確認。情報源の選び方・引用形式は「情報源ガイドライン.md」に従う。
- note 記事の末尾に「参考情報源」欄を設ける(発行元・URL・閲覧日)。出典を示せない数字は書かない。
- **[検索型]の週の note 記事には「SEO ブロック」を付ける**:①想定検索キーワード(主1+関連2〜3)②タイトル案への主キーワード前方配置③見出し(H2)へのキーワード配置④FAQ 3問(検索者が次に調べる質問を Q&A 形式で本文末尾に)⑤note ハッシュタグ案5個。導入の200字以内で検索意図への答えを先出しする。
- 現場が特定できる写真・映像は使わない(撮影は事務所・倉庫・車内)。
- 固有名詞(取引先・契約内容の詳細)は伏せ、一般論＋体験談の形にする。守秘条項に抵触しないか公開前に必ず確認。

## 週次インタビュー(素材出し)の型

毎週、生成前に圭佑が以下に15分で答える(音声入力で OK):

1. 今週テーマの実体験エピソードをひとつ(数字・情景・セリフ付きで)
2. そのとき一番感情が動いた瞬間
3. 読者(施主 or 同業後継者)に持ち帰ってほしいことをひと言

この3つの回答をそのまま Claude Code に渡せば、テーマパックに織り込んで生成する。

## 生成指示テンプレ(Claude Code に貼る)

> 「第N週のテーマパックを生成。インタビュー回答:(上記3問の回答を貼付)。出力:①note 記事下書き(タイトル案3本+参考情報源欄付き)②リール台本2本(各30〜45秒、フック・本編・CTA 構成)③カルーセル構成案(表紙+5枚+CTA)④ストーリーズ告知文。共通ルール・PR 表記規定・情報源ガイドライン.md に従うこと」
